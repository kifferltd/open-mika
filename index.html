Open Mika

Open Mika is an open-source Java runtime dedicated to supporting the OSGi RFC 26 Foundation Execution Environment. It is characterised by a small footprint both in terms of the executable size of the runtime itself (<2MB for both VM and class libraries, in the “headless” version) and in terms of the external libraries used (libc and on Linux libpthread). The BSD-style licence is equally lightweight in the obligations it places on the redistributor; the source code may be modified or extra libraries may be linked in without the obligation to provide source code to end users.

Because of the deliberate restriction to OSGi RFC 26, currently Mika does not support Java 5 language features such as generics. A version with Java 5 support is in preparation, but initially this will only be available as part of the Mika Max product. Support is provided for JNI on Linux and on those uClinux targets which support the use of shared libraries.

A number of features can be selected at the time of building Mika: by omitting unneeded features a smaller (and in some cases faster) Mika binary can be created.

AWT options
  • None - no AWT is provided.
  • Framebuffer - writes directly to the Linux framebuffer device
  • X11 - opens an X11 frame and uses it as a “virtual framebuffer” (primarily intended for development purposes).
Jarfile verification options
  • None - no verification is performed.
  • Java - Jarfile verification is performed in Java code.
  • Native - Jarfile verification is performed in Native code (not yet available).
Bytecode options
  • False - no verification is performed.
  • True - Bytecode verification is performed.
Security options
  • None - no security checks are performed.
  • Coarse - security checks are performed by invoking a user-supplied security manager.
  • Fine - security checks are performed by directly invoking the built-in AccessController.
Optional libraries
  • java.beans - a minimal implementation suitable for use on an embedded client machine. For a full implementation it is better to adopt code from another project such as Apache Harmony.
  • javax.comm - an implementation for Linux/uClinux, covering serial ports only (no parallel port support).

Supported platforms

The standard Mika codebase can be compiled and run directly on the following platforms, using a suitable gcc-based cross-compiler:

  • CPU architectures: ARM (arm7tdmi, arm9, xscale; ...), MIPS (e.g. bcm63xx), PowerPC, x86 (32-bit mode only).
  • Operating Systems: Linux, uClinux.

Porting to other 32-bit CPU architectures and operating systems is certainly possible, but you would be well advised to contact KIFFER for support.

License

The Mika-specific source code is covered by a BSD-style license, which allows modified versions to be distributed without an obligation to supply source code. Some classes are derived from the GNU Classpath project, and these are covered by the Classpath “GPL with linking exception” license. Other classes are derived from the Apache Harmony project and are covered by ASL 2.0.

Our aim is that Mika can safely be linked with native code libraries without endangering confidential or proprietary information that these may contain.
